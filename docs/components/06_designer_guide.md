# 06. Paoji 设计师指南

> English version: [../en/components/06_designer_guide.md](../en/components/06_designer_guide.md)

这份文档给设计师使用，重点说明如何设计 Paoji 组件和表情，而不是底层代码。

## 1. 设计师需要理解的三个对象

| 名称 | 你可以把它理解成 |
|---|---|
| PJD | 一个表情的设计源文件 |
| PJP | 一套可复用的组件库 |
| YS Token | 用户复制和分享的短码 |

## 2. 组件类型

常见组件类型：

```text
base       脸型 / 身体基底
eye        眼睛
brow       眉毛
mouth      嘴巴
cheek      腮红
deco       装饰
effect     特效
material   材质
motion     动画
background 背景
foreground 前景
```

## 3. 标准画布

建议所有组件采用：

```text
viewBox = 0 0 1024 1024
```

这样方便跨组件对齐。

## 4. 锚点设计

基底组件必须提供锚点：

```json
{
  "anchors": {
    "leftEye": { "x": 400, "y": 455 },
    "rightEye": { "x": 624, "y": 455 },
    "mouth": { "x": 512, "y": 615 },
    "leftCheek": { "x": 350, "y": 560 },
    "rightCheek": { "x": 676, "y": 560 }
  }
}
```

锚点的意义是：换脸型后，眼睛、嘴巴、装饰还能自动找到正确位置。

## 5. 组件设计要求

一个好组件应该：

- 有清晰用途。
- 可以复用。
- 有参数，但参数不要过多。
- 有官方 fallback。
- 不依赖外部图片。
- 不使用脚本。
- 适配 1024×1024 画布。
- 小尺寸仍然可辨认。
- 有明确语义标签。

## 6. 视觉风格建议

Paoji 官方基础风格应保持：

- 简洁
- 高识别
- 柔和
- 可缩放
- 可组合
- 小尺寸清晰
- 不依赖复杂阴影才能成立

用户包可以有自己的风格，但必须提供 fallback。

## 7. 设计流程

```text
1. 选择组件类型
2. 在 1024 画布中绘制
3. 设置锚点
4. 设置参数
5. 选择 fallback
6. 填写语义标签
7. 生成预览
8. 打包为 PJP
9. 生成测试 Token
10. 检查降级显示
```

## 8. 不建议设计的组件

- 只能在超大尺寸看清的组件
- 没有 fallback 的组件
- 依赖外部字体或外部图片的组件
- 视觉风格过度复杂且无法降级的组件
- 同名语义混乱的组件
