Open emoji object protocol.

⟦YS1🐱💜🥺😿🫧🌙🪽💧~AgEBAf8A.gGs⟧

Paoji 不是更多 emoji，而是一套允许表情无限扩展、被验证、被渲染、被降级显示的对象协议。

一句话：官方组件保证最低可见性，用户组件保证无限创造。

02YS Token

⟦YS1<visualHint?>~<payload>.<crc>⟧
字段含义
YS协议族，大小写敏感。
visualHint视觉提示区，不是权威数据。
payload机器入口数据，可为 InlineCore / PackRef / DocRef。
crc校验，防止误替换和篡改。

03PJD / PJP / PJR

PJDdocument

表情明文文档，描述节点、组件、语义、能力需求和 fallback。

PJPpack

资源包，承载组件、材质、动画、角色模板和预览。

PJRregistry

注册表，负责命名空间、版本、hash、信任等级和发现。

Rendererruntime

负责解析、加载、验证、沙箱渲染和降级显示。

04组件生态

官方组件也会大量增长，所以官方组件和用户组件必须使用同一套 PJP 格式。区别不是格式，而是 trust level。

paoji.core.eye.sleepy
paoji.std.deco.bubble
paoji.official.cloud.deco.soft-bubble
user.hiyunshu.cloudglass.eye.sleepy-crystal
creator.mochi.plushcat.mouth.pout

05设计师规则

  • 使用 1024×1024 标准画布。
  • 组件必须声明类型：base / eye / mouth / deco / material / motion。
  • 基底组件必须提供锚点。
  • 复杂组件必须有官方 fallback。
  • 不要依赖脚本、外部图片、远程字体或不可控滤镜。

06降级策略

完整 PJP 渲染
  ↓
基础 PJD 渲染
  ↓
Paoji Core fallback
  ↓
Visual Hint
  ↓
alt 文本

这是 Paoji 替代 emoji 的生命线：不能完整显示，也必须尽量有东西可见。

07多语言与授权

Paoji 的文档与生态支持中英双语（zh-CN / en-US）。协议字段、Token、namespace、hash、version 不本地化；UI 文案、组件显示名、alt 文本、设计文档可以本地化。

支持语言：zh-CN, en-US
协议授权：MIT
组件包授权：由各 PJP Pack 自行声明

协议本体采用宽松 MIT 授权；用户组件包、商业组件包或第三方视觉资产可以声明独立 license。
