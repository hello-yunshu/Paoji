告别固定列表，拥抱无限表达。

Paoji 是一套开放式表情对象协议：让任何人都能创造、分享、解析、渲染和降级显示新表情。

它用 YS Token 传播入口，用 PJD 描述表情结构，用 PJP 承载官方与用户组件，用 Registry 管理命名、版本、信任和回退。

001Protocol

短码只是入口，表情是对象。

⟦YS⟧

YS Tokenshort entry

用于传播、识别和校验。它不是完整创作文件，而是一个可以被解析器展开的入口。

→ expand
PJD

Documentexpression doc

完整表情明文态，包含节点、组件引用、语义、能力需求和降级规则。

→ resolve
PJP

Packresource pack

组件、材质、动画、角色模板和 fallback 的资源包。官方和用户组件使用同一格式。

→ compose
🌙🪽💧✨

Visual hint is fallback, not truth.

视觉提示区让人一眼看懂，也能在无渲染器环境下降级显示。但真正的权威数据永远来自 payload 展开的 PJD 或资源包引用。

002Component Ecosystem

官方会增长，用户会失控，所以一开始就包化。

Paoji Coretrust · core

最小核心。所有渲染器必须支持，保证最低可见性。

Paoji Standardtrust · std

稳定扩展。数量可以增长，但必须长期兼容。

Official Packstrust · official

官方主题包。按需下载、校验 hash、缓存渲染。

User Packstrust · user

用户和创作者的无限扩展。必须命名、签名、降级和沙箱化。

003Designer Guide

设计组件，而不是画死图。

组件需要提供什么？

typebase / eye / mouth / deco / material / motion 等标准类型。
anchors锚点让部件可以自动对齐，比如 leftEye、rightEye、mouth。
params可调参数，如颜色、透明度、闪光强度、材质程度。
fallback复杂组件必须能退回到官方核心组件。
